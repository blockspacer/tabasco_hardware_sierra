45 /*
46  * Name: mmpool
47  *
48  * Purpose: This definition is provided so that users can allocate
49  * sufficient storage for their memory pool structures.
50  * However, users are forbidden from manipulating the
51  * contents of this structure and can only pass a pointer
52  * to it into MM package global functions when
53  * required.
54  *
55  * Members: mmnextp - Pointer to the pool header structure of the
56  * neighbouring pool (optional for debugging)
57  * mmfreep - Pointer to the free block list.
58  * mmnamep - Pointer to a NULL-terminated character string
59  * containing the name of this pool
60  * mmvalid - A sentinel value that indicates whether the
61  * pool has been initialized or not
62  * mmnumbks- Number of blocks in the pool
63  * mmblocksz - Size of the User-portion of each block
64  * mmfirstp - Address of the first block for debugging purpose
65  * mmlastp - Address of the last block in the pool
66  * mmsemaph - Semaphore used to make accessing a pool a thread-
67  * safe operation
68  * mmblog - flag to mark pool for logging
69  *
70  * Notes: The free list is implemented as a singly-linked list
71  * so that the "mmfreep" points to the *last* block in
72  * the list and the last block always points to the
73  * first block in the list.
74  *
75  * The "mmbufhdr" pointer is defined within the mmidefs.h
76  * file and external packages don't need to know about it
77  *
78  */
