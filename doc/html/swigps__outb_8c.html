void respGetNmeaPort ( unsigned char *  bufp,
swigpsMsgType  type,
size_t  length,

)

Response handler for receiving the NMEA serial port handle from the swiril side.

Parameters
[in]bufpPointer to storage containing the response packet
[in]typeThe value from the incoming packet header which triggered a call to this function
[in]lengthThe total length of the variable portion of the response packet
[in]uparmarbitrary void * value. Contains a pointer to the state variable for the main state thread in swigps.c
Returns
none
Abort:
none
Note
This function is only called if there was no error code contained in the incoming packet header.

Other than storing the GPS port name into the state structure and marking the name as initialized, this function has no other effect.

The NMEA port handle is a key component in whether this Vendor- specific GPS code will allow its "start()" interface function to be called. If the handle is unknown at the time of the start() call then an error is returned to the interface's owner.

Furthermore, the ENGINE ON status is not sent to the interface's owner until the GPS handle is returned. A properly designed GPS Location Service ought not to call any more interface functions until the ENGINE ON status is received.

int swigps_getnmeaport ( GPS_RILTYPE  riltype,
int  rilgpsfd 
)

Ask for the NMEA USB port name from the swiril side.

This function builds and sends a request to the swiril_gps side (Outbound direction).

Parameters
[in]riltypeThe RIL type expected to be in use by the GPS vendor specific code. Used by the ril-daemon for validation purposes
[in]rilgpsfdFile descriptor for the socket connecting swigps to swiril_gps
Returns
0 - The operation was successful < 0 - An error occurred: -ENOMEM - Insufficient storage (returned from subroutine call) -EINVAL - Internal failure (Abort would be appropriate)
Abort:
none
Note
This function is a key component in this swigps vendor-specific startup. The interface's "start()" function will return with an error until the NMEA port handle is known and the ENGINE ON status will only be returned when this function's counterpart, respGetNmeaPort() has been called with a valid handle. Therefore, calling this function is mandatory and should happen very early on in the main gps state thread's startup.
