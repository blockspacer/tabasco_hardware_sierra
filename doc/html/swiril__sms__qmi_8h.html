RIL_REQUEST_CDMA_SMS_ACKNOWLEDGE.

Acknowledge successful or failed receipt of SMS previously indicated via RIL_UNSOL_RESPONSE_CDMA_NEW_SMS

Protect the entire function - need to make sure that any new incoming SMS (in callback thread) is not serviced before treating the current ACK

Received a valid ACK, this variable would be updated to true by NewSMSDispatch(), if there is an SMS to be dispatched.

If the previous message was not successfully acked, try re-sending it again later on reception of a New SMS.

The stored message from the beginning of the queue (for which the ACK is received) is deleted from Device on successfull Ack

RIL_REQUEST_SMS_ACKNOWLEDGE.

Acknowledge successful or failed receipt of SMS previously indicated via RIL_UNSOL_RESPONSE_NEW_SMS

Protect the entire function - need to make sure that any new incoming SMS (in callback thread) is not serviced before treating the current ACK

Received a valid ACK, this variable would be updated to true by NewSMSDispatch(), if there is an SMS to be dispatched.

The previous message was not successfully acked, try re-sending it again later on reception of a New SMS. If there are any pending class 0 messages, send it on priority and reprioritize the queue

The stored message from the beginning of the queue (for which the ACK is received) is deleted from Device on successfull Ack
