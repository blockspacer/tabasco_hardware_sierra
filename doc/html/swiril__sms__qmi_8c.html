Create storage area, and store the information of the received SMS in the queue

Message mode is mandatory, hence setting this message mode even if this is not received from the device

Check to find whether the message is class 0 message, and if so needs to be treated in priority and has to be inserted into the beginning of the queue. This has to be only taken care for GSM messages now

Treat message class 0 with highest priority, and enqueue to the beginning of the queue if there is no outstanding ACK waiting, or second of the queue if there is a wait for the ACK of the previous SMS

bool parse_and_prioritize_queue ( )

Prioritize the message queue to push all the class 0 messages to the beginning of the queue.

Parameters
none
Returns
true if re-prioritized, else false
Note
Need to aquire a mutex before this function is invoked.

If the first element present in the queue is a class 0 message the queue is already prioritized, hence we return false

If the second element present in the queue is a class 0 message the queue needs to be prioritized, push all the class 0 messages to the beginning of the queue and return true so that the latest class 0 message is dispatched

If there are class 0 messages present and are not in front of the queue, the class 1 message on top has to be inserted to the end of the class 0 messages in the queue
